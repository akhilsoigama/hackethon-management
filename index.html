Digital Nabha Shiksha

Bridging the Digital Divide in Rural Education with Optimized Technology

The Challenge

Government schools in Nabha and surrounding rural areas face significant digital infrastructure challenges:

Lack of Infrastructure

Outdated computer systems with limited functionality

Connectivity Issues

Unreliable or non-existent internet access

Resource Scarcity

Limited access to quality digital educational content

Impact on Students

  • Limited academic growth and future employability
  • Widening gap between rural and urban education standards
  • Lack of digital literacy skills essential for modern workforce
  • Perpetuation of educational and economic disadvantage cycles

Our Approach

Digital Nabha Shiksha provides a comprehensive solution tailored for rural education needs:

Offline-First Design

Works without internet connectivity

Local Language Support

Available in Punjabi and Hindi

Low-End Optimized

Runs smoothly on basic smartphones

Technical Architecture

  • Frontend: React with Vite for fast development and optimized builds
  • Backend: AdonisJS for a structured and maintainable API
  • Data Fetching: SWR for efficient caching and state management
  • Offline Capability: PWA technology with service workers
  • Storage: IndexedDB for offline data persistence

User Interface Design

Our UI is designed for simplicity, accessibility, and optimal performance on low-end devices:

Student Dashboard

Simple navigation with large buttons and clear typography

Lesson View

Minimalist design with focus on content

Teacher Portal

Clean progress tracking and class management

Design Principles

  • Simplicity: Minimal UI elements to reduce cognitive load
  • Consistency: Uniform design patterns throughout the app
  • Accessibility: High contrast colors and large touch targets
  • Performance: Optimized images and lazy loading of content
  • Offline-First: UI clearly indicates offline/online status

Performance Optimization

We've implemented multiple strategies to ensure smooth performance on low-end devices:

Code Splitting

Lazy loading components with React.lazy() and Vite's dynamic imports to reduce initial bundle size.

Efficient Data Fetching

Using SWR for smart caching, background updates, and efficient revalidation of data.

Asset Optimization

Images are compressed, converted to WebP format, and lazy loaded to reduce bandwidth usage.

Memory Management

Efficient component rendering with memoization and avoidance of memory leaks.

Loading Strategies

  • Critical CSS: Inlined to prevent render-blocking
  • Prefetching: Anticipating user needs and preloading resources
  • Caching: Service worker caching for instant loading of previously visited pages
  • Bundle Analysis: Regular monitoring of bundle size to prevent bloat

Library Recommendations

Choosing the right libraries is crucial for performance on low-end devices:

Avoid These Libraries

  • Moment.js - Too heavy, use date-fns or lightweight alternatives
  • Lodash - Import only specific functions, not the entire library
  • jQuery - Not needed with modern browser APIs
  • Bootstrap - Heavy, use lightweight CSS alternatives
  • Axios - Use native fetch with a lightweight wrapper

Recommended Libraries

  • React Hook Form - Lightweight form management
  • date-fns - Modern date utility library
  • Zustand - Lightweight state management
  • React Query/SWR - Efficient data fetching
  • Framer Motion - Performant animations

Bundle Size Strategy

  • Regularly audit bundle size with tools like Webpack Bundle Analyzer
  • Set performance budgets (e.g., max initial bundle size of 100KB)
  • Use tree-shaking to eliminate unused code
  • Prefer smaller alternatives to popular libraries
  • Implement code splitting for larger features
